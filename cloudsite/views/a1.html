Microservices Architecture

 

Es una arquitectura que busca cambiar la forma común de desarrollar software monolítico, es decir aquellos sistemas que tienen un relación consigo mismos en donde para modificar o actualizar alguno de sus procesos se requerirá cambiar también otras secciones y rediseñar incluso parte de su estructura.

Por otra parte en la arquitectura de microservicios cada proceso se toma como independiente con su propia funcionalidad como un servicio, es decir que incluso cada uno de lo procesos dentro del software pueden ser distintos lenguajes de programación, así mismo distribuir los servicios entre servidores, replicando cada uno de estos a preferencia.

No existe una definición completamente definida pero se puede describir por las características que se busca cumplir en esta arquitectura. Entre estas caracteristicas las mas importantes son:

  • Componentization : Esto mediante múltiples componentes que son unidades de software independientes, reemplazables y actualizables.

  • Services: Son componentes fuera de un proceso los cuales se comunican con un mecanismo como un webrequest o calls. Esto se utiliza en vez de solo librerías. Ya que los servicios pueden correr independientemente.

  • Products not Projects:  Esta arquitectura evita el modelo típico de vender un software y dejarlo olvidado. “You build it , you run it”

  • Esta arquitectura también cuenta con características para aprovechamiento de recursos,  que la aplicación está diseñada para mantener errores y corregirlos.

 

Routing in AngularJS

 

$route es servicio que permite el manejo de vistas en angular, este servicio te deja controlar el enrutamiento de cada una de las vista en la misma aplicación “Single Page Application”.

Lo explicare en este ejemplo:

Para la aplicación [app], se le agrega su módulo ngRoute como dependencia.

var app = angular.module("app", ["ngRoute"]);

De esta forma la aplicación tiene acceso al $routeProvider

app.config(function($routeProvider) {

   $routeProvider

   .when("/", {

       templateUrl : "views/main.htm",

Controller: "tu controlador"

//De esta forma se declara que el template de mi vista está en el folder views en el      archivo main.html y que tiene su propio controlador que puede ser el mismo o diferente para cada view.

   .when("/red", {

       templateUrl : "views/second.htm"

Controller: "tu controlador"

   })

   

});

 

En el index.html es importante que se agregue en el inicio del los tags ya sea en el <html> o el body lo siguiente

<body ng-app="myApp">

Esto para  que la aplicación este conectada con el index.html. Por otra parte lo que permitirá router las vistas en base a su controlador es:

<div ng-view></div>

De esta forma cuando se quiera cambiar de vista simplemente podemos utilizar las funciones que incluye routeProvider  

$location.path("/TuView");

 

 

Multiple Access keys AWS

Se pueden generar otras access keys definiendo nuevos perfiles de aws con el comando

Aws configure -profile [Username]

De esta forma cada uno de los usuarios tendrá sus propias keys

 

[user1]

aws_acces_key = 8834JEWRHSK8934

Aws_secret_acces_key = 8834JEWRHSK8934

 

[user2]

aws_acces_key = 8834JEWRHSK893

Aws_secret_acces_key = 8834JEWRHSK8934
