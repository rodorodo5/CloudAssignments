4. Read and explain the following article by Martin Fowler: http://martinfowler.com/ articles/richardsonMaturityModel.html

En este articulo Martin Fowler habla del modelo de Leonard Richardson. En donde explica los niveles del modelo Maturity para llegar a una explicación concreta del uso de tecnicas de Restful API's, el cual divide en 4 niveles (0-3)

Dentro del nivel 0 se explica que se utiliza el HTTP protocol como un sistema de transporte para interacciónes remotas. En realidad todo este paso no es mas que tunneling de comunicación de respuestas. El cual puede ser XML de responses back and forth.

En el nivel 1, introducir los resources e indentificarlos independientes cada uno de estos recursos mediante su URI, es decir mantener ese recurso para despues postearlo.

En el nivel 2 explica que a diferencia de los niveles 0 y 1 utilizar POST o GETS ya que en estos niveles ambos estan siendo utilizados como mecanismo de comunicacion o transporte, y direccionar las interacciones por medio de HTTP. En cambio el nivel 2 se busca utulizar HTTP verbs tal como serian utilizados en el protocolo. Es decir el utilizar GET como safe operation es crucial, lo cual permite hacer caching.

Por ultimo en el nivel 3 llamado Hypertext As The Engine Of Application State es donde ocurre la magia, y direcciona los URIS y lo define como su resource para poderlo manipularlo. Incluso modificar los URIs sin romper el scheme.

5. Explain why cloud computing is important.

•Price: Cloud Computing permite a las empresas dejar a un lado la inversion en equipo fisico, como server racks, cableado, redes etc. Facilitandoles las necesidades con un click. Sin tener que preocuparse por nada de lo mencionado. El costo reduce considerablemente ya que este equipo es caro y tambien su mantenimiento.


Scalability: Cloud computing te permite una increible escalabilidad, si eres una empresa pequeña que busca recursos adaptables para sus necesidades pero en busqueda de crecer, con cloud computing puedes ir creciendo segun las necesidades, esto te permite escalar y mantener tu misma estructura solo obteniendo los recursos que crees necesarios para ti.


Flexibility: Manejar estos recursos mencionados en donde quieras cuando quieras, con solo tener internet. Faciles de manejar y administrar sin necesidad de pagar mantenimiento fisico.


Automation: Todas estas caracteristicas mencionadas pueden ser automatizadas,es decir, en caso de necesitar mas recursos, puede adaptarse automaticamente a lo que se requiere.


Information security & backups: Toda tu información es segura y es algo de lo cual tu tampoco te preocupas directamente, toda tu información esta lista para cualquier tipo de error o desastre.
